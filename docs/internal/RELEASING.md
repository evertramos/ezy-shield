# Releasing EzyShield

Releases are built by goreleaser (`.goreleaser.yaml`) from a git tag.

## How a release happens

1. **Automatic (tag push):** push a tag `v*` â€” `release.yaml` runs goreleaser

2. **Manual (workflow dispatch):** run the đźŹ·ď¸Ź Release workflow and pick

`rc`/`alpha`/`beta` tags become **prereleases** and never take the "latest"
mark (`release.prerelease: auto`).

## Artifacts per release

| Artifact | Purpose |
|----------|---------|
| `ezyshield-linux-{amd64,arm64}`, `ezyshield-enforcer-linux-{amd64,arm64}` | Raw binaries. **Naming is a contract**: `scripts/get.sh` downloads these exact names and greps them in `checksums.txt`. Never rename without updating `get.sh` in the same PR. |
| `ezyshield_<ver>_linux_{amd64,arm64}.tar.gz` | Both binaries + LICENSE/README for manual installs |
| `ezyshield_<ver>_linux_{amd64,arm64}.{deb,rpm}` | Distro packages (nfpm): binaries in `/usr/bin`, units in `/usr/lib/systemd/system`, config **examples** in `/etc/ezyshield/*.example` (noreplace), postinstall creates the `ezyshield` user/group and 0750 dirs, **never enables/starts units** |
| `checksums.txt` | SHA-256 for everything above |

## Package/system layout notes

- The units inside packages are generated by
  `scripts/package/mk-package-units.sh`: identical to `configs/systemd/`
  (single source of truth for hardening directives) with `ExecStart`
  rewritten `/usr/local/bin` â†’ `/usr/bin`. Script installs (`get.sh`) keep
  using `/usr/local/bin`.
- Maintainer scripts: `scripts/package/postinstall.sh` (user/group + dirs +
  conditional `daemon-reload` + next-steps hint) and `preremove.sh` (stop
  units if active; **never** deletes `/var/lib/ezyshield`).

## Local testing (never publishes)

```bash
goreleaser check                       # validate config
goreleaser release --snapshot --clean  # full build into dist/
# install-verify like CI does:
docker run --rm -v "$PWD/dist:/dist:ro" debian:12 bash -ec \
  'apt-get -qq update >/dev/null; apt-get -qq install -y systemd >/dev/null; dpkg -i /dist/ezyshield_*_linux_amd64.deb; ezyshield version'
```

The `packages` job in `ci.yaml` runs the same snapshot + container
install-verify (debian:12 for .deb, rockylinux:9 for .rpm) on every PR, so a
broken package config fails before a tag ever exists.
